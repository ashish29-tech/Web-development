useRef(): Ek aisa hook...tab use hota hai jab DOM ke andar changes show karne ho..
ex: predefined text ko change karna hai
usecase1: ye hook hai..means function...DOM ke andar jitne elements(Div, h1, strong naam se bna liya etc.) hote hai..uska refernce mil jata hai..so we can manipulate it
usecase2: state maintain karne m kaam ata hai..state hamare case m variable. state ko maintain karne ke kaam ata hai between 2 renders
ex: 
Render-1..App first time render hua
And state m koi change kiya...and state ka change component catch kar leta hai
suppose in 2 render ke beech koi state hai ham use maintain karana chahte hai...toh ye kaam ye useRef() hook kar sakta hai
state -> preserve/maintain state..means pehle render ke time 1 thi toh render 2 ke time bhi 1 maintain rahegi
Render-2..2nd time render hua

Now v-important concept...
Routing... means /products, /reviews/:id pe jana etc.
React ke paas khud ka router nahi hai...means react ke paas aisa kuch bhi nahi hai ki ham routing perform kar sake
React 3rd party library ka use karta hai jise...react-router-dom kehte hai... https://reactrouter.com/home
3rd party library ka use karta hai Toh ise install karna padega


-Routing use karne ke baad bhi SPA honi chahiye
- Client side bundling honi chahiye
- Client side rendering chahiye

We all know react is a library. Library js ka data mang rahi hai...and data ko dom se manipulate kar dete hai. Page refresh hota hai ? No..toh sari chize page create ho rahe horahe hote hai wo client ki side pe ban rahi hogi.
Client side bundling: Ek baar server call hota hai...and sara js bundle hoke aa jata hai..toh jaise /(slash) routing wala data...apne aap bifurcate hoke chala jata hai..dobara server call karne ki jarurat nai padti..

Client side rendering: jo page create ho rha hai...wo sara page js ki madad se dom manipulate se ho rha hai...and page refresh nai hoga...
